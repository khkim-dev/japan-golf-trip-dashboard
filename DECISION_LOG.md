# Decision Log

## 2026-07-05

| 결정 | 이유 |
| --- | --- |
| Vite + React를 사용한다 | 모바일 UI, 상태 관리, 정적 배포 학습에 적합하다 |
| GitHub Pages 배포를 v1.0 범위에 포함한다 | Commit, Push, 배포, 카카오톡 공유 흐름을 한 번에 학습할 수 있다 |
| v1.0에서는 mock data만 사용한다 | 개인정보와 외부 연동 리스크를 줄이고 MVP에 집중한다 |
| Memory Mode는 v1.1 이후로 분리한다 | 첫 버전은 여행 준비와 공유 가능한 대시보드에 집중한다 |
| 프로필 이미지 파일명은 영문 슬러그를 사용한다 | GitHub Pages 배포 시 URL 인코딩 문제를 줄이기 위해서다 |
| `localStorage`는 핵심 저장 방식으로 사용하지 않는다 | 개인 브라우저에만 저장되어 카카오톡 단체방 공유 현황판 목적과 맞지 않는다 |
| v1.1에서 Google Sheets 같은 공유 데이터 소스를 검토한다 | 여러 명이 같은 최신 데이터를 볼 수 있는 구조가 필요하기 때문이다 |

## 2026-07-08

| 결정 | 이유 |
| --- | --- |
| Members는 카드형 React UI를 유지한다 | 사진, 이름, 역할을 빠르게 확인하는 목적에는 카드형이 가장 안정적이다 |
| 멤버 포스터형 UI는 실험용 백업으로만 유지한다 | 모바일에서 배치가 어색하고 겹침 위험이 있어 기본 화면으로 적합하지 않다 |
| Schedule 독립 탭을 제거하고 Overview로 통합한다 | Overview가 항공, 일정, 예약 요약을 충분히 담고 있어 모바일 탭 수를 줄이는 편이 더 명확하다 |
| Booking은 React UI로 유지한다 | 숙소/골프장 정보, 지도 링크, 실제 예약 확인이 중요하기 때문이다 |
| Balance는 React UI로 유지한다 | 지출 입력, 멤버 선택, 계산 기능이 필요하기 때문이다 |
| Balance 화면명을 The 19th Hole로 변경한다 | 단순 정산 기능이 아니라 골프 여행의 대표 대시보드라는 인상을 주기 위해서다 |
| 정산 계산 로직을 Utility로 분리한다 | UI와 계산 엔진을 분리해 정확성과 유지보수성을 높이기 위해서다 |
| Yardage Book 탭을 추가한다 | 캐디 없는 일본 골프 라운드에서 홀별 Par, 거리, 공략 메모를 빠르게 확인하기 위해서다 |
| 홀 이미지는 실제 사진 대신 통일된 일러스트 스타일로 구현한다 | 두 골프장 이미지 톤을 맞추고 저작권/출처 리스크를 줄이기 위해서다 |
