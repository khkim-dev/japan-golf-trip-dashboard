# Decision Log

## 2026-07-05

| 결정 | 이유 |
| --- | --- |
| Vite + React를 사용한다 | 모바일 UI, 상태 관리, 정적 배포 학습에 적합하다 |
| GitHub Pages 배포를 v1.0 범위에 포함한다 | Git Commit, Push, 배포, 카카오톡 공유 흐름을 한 번에 학습할 수 있다 |
| v1.0에서는 mock data만 사용한다 | 개인정보와 외부 연동 리스크를 줄이고 MVP에 집중한다 |
| Memory Mode는 v1.1 이후로 분리한다 | 첫 버전은 여행 준비와 공유 가능한 대시보드에 집중한다 |
| 프로필 이미지 파일명은 영문 슬러그를 사용한다 | GitHub Pages 배포 시 URL 인코딩 문제를 줄이기 위해서 |
| `localStorage`는 핵심 저장 방식으로 사용하지 않는다 | 개인 브라우저에만 저장되어 카카오톡 단체방 공유 현황판 목적과 맞지 않는다 |
| v1.1에서 Google Sheets 같은 공유 데이터 소스를 검토한다 | 여러 명이 같은 최신 데이터를 볼 수 있는 구조가 필요하기 때문이다 |
